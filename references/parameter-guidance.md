# 参数指南

根据任务的实际要求推荐参数。不要声称更高设置普遍更好。

## 推理强度

- `low`：简单转换、提取、分类，或答案路径清晰且对延迟敏感的工作。
- `medium`：普通编码、分析、规划和工具使用的均衡默认设置。
- `high`：复杂调试、多来源综合、困难规划或多个相互作用的约束。
- `xhigh`：异常困难且质量优先的推理任务，并且额外成本有充分理由。
- 不要全局推荐 `max`。仅在有具体质量理由的例外任务中提及。

推荐更高推理强度前，检查提示词是否缺少成功标准、依赖规则、工具路径、证据规则或验证闭环。

## 详细程度

- `low`：简短回答、简单转换、分类和紧凑的操作性响应。
- `medium`：解释、编码任务、审查和普通报告的默认设置。
- `high`：详细研究综合、规格说明、审计、教程，或价值依赖细节保留的产物。

先保留必需事实、决策、注意事项、证据和后续行动，再删减引言、重复内容、安抚性措辞、示例或可选背景。

## 条件性建议

- 仅对密集、大尺寸、对 OCR 敏感或对坐标敏感的图像推荐原始图像细节。
- 仅在具有明确模式和交接步骤的有界确定性归约阶段推荐程序化工具调用。
- 不相关工具会增加路由歧义时，建议减少暴露的工具数量。
- 将不受支持或未核验的参数标记为 `未核验`；不要猜测 API 是否可用。
