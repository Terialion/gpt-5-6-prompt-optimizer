# GPT-5.6 Prompt Optimizer

将自然语言请求或现有 Prompt 转换为更适合 GPT-5.6 / GPT-5.6 Sol 的中文 Prompt，并同时给出简短的优化说明、假设、缺失信息和建议参数。

这是一个面向 Codex 的本地 Skill，适合在你准备把一个模糊请求交给 GPT-5.6 之前，先把目标、约束、输出格式和验收标准整理清楚。

> 本项目是社区 Skill，不代表 OpenAI 官方产品、官方模型配置或官方 Prompt 模板。模型名称、能力和参数支持情况可能随平台变化。

## 为什么需要这个 Skill

很多原始请求并不是“信息太少”，而是关键信息没有以模型容易执行的形式组织起来。常见问题包括：

- 目标、背景、约束和输出物混在一段话里；
- 没有明确完成标准，模型不知道什么算做对；
- 缺少必要信息时，模型要么擅自猜测，要么先问一长串问题；
- 没有区分事实、用户偏好、证据要求和授权边界；
- 没有根据任务复杂度选择 reasoning effort 和 verbosity；
- 原始 Prompt 已经足够好，却被过度改写，导致语气、格式或事实发生漂移。

这个 Skill 的作用不是把每个请求都改写成很长的“超级 Prompt”，而是识别真正影响结果的行为要求，保留用户意图，并用最小必要结构提高执行确定性。

## 核心能力

- 识别请求的真实意图、可见结果、任务类型和成功标准；
- 把目标、上下文、证据、约束、权限和输出契约分开组织；
- 保留用户明确给出的事实、数字、文件名、格式、语言和限制；
- 在缺少信息时做最小且可见的合理假设，不因为非关键缺口阻塞生成；
- 根据任务复杂度建议 `reasoning effort` 和 `verbosity`；
- 区分研究、编码、编辑、规划、审查、工具使用和冲突 Prompt 等常见意图；
- 生成中文 Prompt，并提供简洁的中文诊断信息；
- 在最终输出前检查事实保留、范围漂移、缺失门槛、参数匹配和结构完整性。

## 什么时候使用

当你希望优化、改写、迁移或审查一个 GPT-5.6 Prompt 时使用，例如：

- “把这个需求改成适合 GPT-5.6 的 Prompt”；
- “优化这段代码审查 Prompt，但不要改变原意”；
- “把这个模糊任务整理成可执行的研究 Prompt”；
- “告诉我这个 Prompt 缺了什么，并建议 reasoning effort 和 verbosity”；
- “这个 Prompt 已经很长了，帮我删掉冗余但保留约束”。

如果用户只是希望直接完成任务，而不是优化 Prompt，不应强行把普通任务转换成 Prompt 优化报告。

## 使用方式

在 Codex 中直接调用：

```text
Use $gpt-5-6-prompt-optimizer to optimize the following request for GPT-5.6:

帮我分析这个项目最近的登录问题，修复它并告诉我改了什么。
```

也可以直接用中文提出请求：

```text
帮我把下面这段需求改成适合 GPT-5.6 的高效 Prompt。
不要改变事实和输出语言，并列出缺失信息和建议参数：

[粘贴原始需求]
```

若用户没有明确要求某种模型语言，优化后的 Prompt 默认使用中文；询问用户时仍使用中文。

## 输出格式

每次优化严格返回四个部分：

### 1. 优化后的 Prompt

放在一个 `text` 代码块中，内容应可直接复制使用。只加入会改变下游行为的指令，不暴露隐藏推理，也不凭空添加工具、API、事实或结果。

### 2. 优化说明

用两到五条简短说明解释最重要的行为变化，例如：

- 明确了目标和完成标准；
- 将事实、约束和输出格式分开；
- 增加了证据边界或停止条件；
- 删除了互相冲突或不影响行为的冗余指令。

### 3. 假设与缺失信息

分别列出：

- **假设**：为了让 Prompt 可执行而采用的最小假设；
- **缺失信息**：可能影响结果、但当前不阻塞生成的未知信息。

如果没有实质性缺口，明确写“无重要缺失信息”。

### 4. 建议参数

至少说明：

```text
reasoning effort: low | medium | high | xhigh
verbosity: low | medium | high
其他参数: 仅在确实相关时给出
```

参数建议必须与任务复杂度匹配，不默认把所有任务都设为最高等级。

## 处理流程

Skill 内部按以下顺序工作：

1. 解析用户动作、期望结果、领域、输入材料、约束和授权边界；
2. 识别相关意图配置，不要求用户先选择任务类别；
3. 保留事实、数字、标识符、文件名、语言、格式和明确限制；
4. 删除重复、空泛、互相冲突或不会改变行为的指令；
5. 构建以结果为先的中文 Prompt；
6. 检查成功标准、依赖关系、工具边界、证据要求和停止条件；
7. 根据任务风险和复杂度建议参数；
8. 按固定四段格式输出。

缺失信息不会自动阻塞 Prompt 生成。只有当缺口会让目标、权限或结果类型无法确定时，才应在假设中明确标注并收紧输出边界。

## 设计来源

本 Skill 的设计来源分为三类，分别对应“为什么这样写”和“如何执行”：

### GPT-5.6 官方提示指导

结构和参数建议参考 OpenAI 面向 GPT-5.6 的提示指导，重点吸收以下原则：

- 先定义结果和完成标准，再添加必要约束；
- 保留用户明确给出的事实和要求；
- 不凭空增加模型能力、工具、API 或外部事实；
- 对缺失信息采用最小、可见的假设；
- 根据任务复杂度选择推理强度和详细程度。

参考：[OpenAI GPT-5.6 model guidance](https://developers.openai.com/api/docs/guides/latest-model?model=gpt-5.6)

### Skill 内部参考规则

仓库中的 `references/` 文件把上述原则落成可执行规则：

- `transformation-rules.md`：请求解析、事实保留、最小假设和行为改写边界；
- `intent-profiles.md`：研究、编码、编辑、审查、规划和工具任务的意图配置；
- `parameter-guidance.md`：`reasoning effort` 与 `verbosity` 的选择依据；
- `output-contract.md`：固定的四段输出格式；
- `quality-checklist.md`：事实、约束、证据、参数和输出结构检查；
- `source-guidance.md`：来源边界和不臆造模型能力的要求。

这些文件是 Skill 的运行依据，不是独立的外部服务或 API 依赖。

### 实际使用反馈

本 Skill 还吸收了实际使用中反复出现的需求：

- 优化后的 Prompt 统一使用中文；
- 优化说明、假设、缺失信息和参数建议必须单独列出；
- 先基于合理假设生成 Prompt，不在生成前强制追问；
- 意图应由请求内容推断，不强迫用户填写任务分类；
- 已经有效的原始 Prompt 只做最小改动；
- 不把隐藏推理过程或未经证实的能力声明写进输出。

## 设计边界

这个 Skill：

- 只优化 Prompt，不自动执行优化后的任务；
- 不替代模型、API、工具或权限配置；
- 不保证任何特定模型版本永远存在或支持所有参数；
- 不会因为使用了 GPT-5.6 名称就自动获得 GPT-5.6 或图像生成能力；
- 不会把 API Key、隐藏推理或用户未提供的敏感信息写进 Prompt；
- 不会在缺少关键权限或证据时伪造完成结果。
